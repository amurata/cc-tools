# タスクファイルテンプレート

## 概要
TFDD（タスクファイル駆動開発）で使用するタスクファイルのテンプレート。

---

## テンプレート

```markdown
# Task-XXX: {タスク名}

## 基本情報

| 項目 | 内容 |
|------|------|
| タスクID | XXX |
| Wave | {wave番号} |
| 優先度 | {high/medium/low} |
| 見積もり | {S/M/L} |
| ステータス | {todo/in_progress/review/done} |
| 担当 | {AIインスタンスID または 未割当} |
| ブランチ | task/XXX_{説明} |
| Worktree | ../_wt-task-XXX-{name} |

## 依存関係

### 前提タスク（これが完了していないと開始できない）
- [ ] Task-YYY: {タスク名}
- [ ] Task-ZZZ: {タスク名}

### 後続タスク（このタスク完了後に開始可能になる）
- Task-AAA: {タスク名}
- Task-BBB: {タスク名}

## 目的
{このタスクで達成すること}

## 受け入れ条件
- [ ] {条件1}
- [ ] {条件2}
- [ ] {条件3}

## 技術仕様

### 対象ファイル
- `path/to/file1.ts` - {変更内容}
- `path/to/file2.ts` - {変更内容}

### 実装方針
{どのように実装するかの概要}

### テスト方針
{どのようなテストが必要か}

## 作業ログ

### {日付} - 開始
- 作業開始
- {実施内容}

### {日付} - 完了
- {最終状況}
- PR: #{PR番号}
```

---

## 使い方

### 1. 新規タスク作成
```bash
# タスクディレクトリに配置
tasks/
├── todo/
│   └── task-XXX.md    # 新規作成はここ
├── in_progress/
├── review/
└── done/
```

### 2. ステータス遷移
```
todo → in_progress → review → done
```
ファイルを対応するディレクトリに移動してステータスを管理。

### 3. 命名規則
- ファイル名: `task-{3桁番号}.md`（例: task-001.md）
- ブランチ: `task/{番号}_{説明}`（例: task/001_add-auth）
- Worktree: `../_wt-task-{番号}-{name}`（例: ../_wt-task-001-auth）

---

## GitHub IDD対応

GitHub Issue駆動の場合は以下の形式を使用：

```markdown
# Issue #{番号}: {タイトル}

## 基本情報

| 項目 | 内容 |
|------|------|
| Issue | #{番号} |
| Wave | {wave番号} |
| Labels | {ラベル} |
| ステータス | {open/in_progress/closed} |
| ブランチ | feature/#{番号}-{説明} |

## 依存関係
- Blocked by: #{依存Issue番号}
- Blocks: #{後続Issue番号}

## 内容
{Issueの説明をここにコピーまたはリンク}

## 受け入れ条件
{Issue の Acceptance Criteria}
```

### コード品質
- [ ] Lint通過

- [ ] 壊れたら気づくテストか？

- [ ] 適切なモック範囲か？

### 完了条件
- [ ] 受け入れ条件すべて満たす
- [ ] 不要コードの削除完了
