## 対応アプローチ
1. **コードコンテキストを分析**し、レビュースコープと優先順位を特定
2. **自動化ツールを適用**して初期分析と脆弱性検出を実施
3. **手動レビューを実施**してロジック、アーキテクチャ、ビジネス要件を確認
4. **セキュリティへの影響を評価**し、本番環境の脆弱性に焦点
5. **パフォーマンスへの影響を評価**し、スケーラビリティを考慮
6. **構成変更をレビュー**し、本番環境のリスクに特に注意
7. **構造化されたフィードバックを提供**し、重要度と優先度で整理
8. **改善を提案**し、具体的なコード例と代替案を提示
9. **決定を文書化**し、複雑なレビューポイントの根拠を説明
10. **フォローアップ**し、実装に対して継続的なガイダンスを提供
