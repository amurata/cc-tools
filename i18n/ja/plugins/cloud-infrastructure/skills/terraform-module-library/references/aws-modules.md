# AWS Terraformモジュールパターン

## VPCモジュール
- パブリック/プライベートサブネットを含むVPC
- インターネットゲートウェイとNATゲートウェイ
- ルートテーブルと関連付け
- ネットワークACL
- VPCフローログ

## EKSモジュール
- マネージドノードグループを含むEKSクラスタ
- IRSA（サービスアカウント用IAMロール）
- クラスタオートスケーラー
- VPC CNI設定
- クラスタロギング

## RDSモジュール
- RDSインスタンスまたはクラスタ
- 自動バックアップ
- リードレプリカ
- パラメータグループ
- サブネットグループ
- セキュリティグループ

## S3モジュール
- バージョニング付きS3バケット
- 保存時暗号化
- バケットポリシー
- ライフサイクルルール
- レプリケーション設定

## ALBモジュール
- アプリケーションロードバランサー
- ターゲットグループ
- リスナールール
- SSL/TLS証明書
- アクセスログ

## Lambdaモジュール
- Lambda関数
- IAM実行ロール
- CloudWatch Logs
- 環境変数
- VPC設定（オプション）

## セキュリティグループモジュール
- 再利用可能なセキュリティグループルール
- イングレス/エグレスルール
- 動的ルール作成
- ルール説明

## ベストプラクティス

1. AWSプロバイダーバージョン ~> 5.0を使用
2. デフォルトで暗号化を有効化
3. 最小権限IAMを使用
4. すべてのリソースに一貫してタグ付け
5. ロギングと監視を有効化
6. 暗号化にKMSを使用
7. バックアップ戦略を実装
8. 可能な場合PrivateLinkを使用
9. GuardDuty/SecurityHubを有効化
10. AWS Well-Architected Frameworkに従う
